Project Implementation

Using tensorflow.js, we were able to convert the model's output into a dense JavaScript model called into our website using a content delivery network.
We also created a user friendly dropdown using the most important features in the model. Users can input overall square footage, square footage that needed heating, square footage that needed cooling and total number of A/C Units. This returns a prediction of the average energy consumption usage for that household by US Census division. Connecting the model on an EC2 instance directly to an S3 folder called by the HTML page allows you to update the current website everytime the model is trained. Another useful S3 property is to enable versioning which allows you to see different versions of the same object helping you to keep track of your objects. We also updated the bucket policy to allow public access to this static website which is the goal here. Another useful future expansion will be to host the static website using S3 and AWS CloudFront. CloudFront uses AWS edge locations to facilitate quick access to your users in other parts of the world. To ensure very high availability, we can keep a backup of our static website in another region because S3 is a regional resource.

Energy consumption data comes from the U.S. Energy Information Administraion (EIA)'s 2015 Residential Energy Consumption Survey (RECS).
